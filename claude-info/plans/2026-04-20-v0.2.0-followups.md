## 1. Dashboard standalone assignment support

**Why:** The backend resolver (`src/utils/assignment-resolver.ts`) exists and CLI commands support standalone. But the dashboard only knows about project-nested assignments — standalone ones at `~/.syntaur/assignments/<uuid>/` are invisible.

### 1.1 Backend — `src/dashboard/server.ts`
- Add `assignmentsDir: string` to `DashboardServerOptions`.
- Thread it through to `createWriteRouter`, `createApiRouter`, scanner, watcher, agent-sessions router, servers router.

### 1.2 Backend — `src/dashboard/api.ts`
- `listAssignmentsBoard(projectsDir, assignmentsDir)` — after scanning project-nested, scan `assignmentsDir/*` for standalone. Each row: `projectSlug: null`, `projectTitle: null`, `projectWorkspace: null`.
- Extract `buildAssignmentDetail(assignmentDir, projectSlug, projectsDir)` helper.
- Add `getAssignmentDetailById(projectsDir, assignmentsDir, id)` that uses the resolver then delegates.
- `getEditableDocumentById(projectsDir, assignmentsDir, documentType, id)` for document routes.
- `getAvailableTransitions(assignmentDir)` — skip dependency checks when standalone.
- `getUnmetDependencies` returns `[]` when `projectSlug === null`.
- `getOverview(assignmentsDir)` — include standalone count in stats; `firstRun = false` once any standalone exists.
- `buildAttentionItems` accepts standalone list; emits `href: /assignments/<id>`, `projectSlug: null`.
- `buildRecentActivity` — same pattern.

### 1.3 Backend — `src/dashboard/api-write.ts`
Accept `assignmentsDir` in `createWriteRouter(projectsDir, assignmentsDir)`. Add these routes:
- `POST /api/assignments` — create standalone. Validate `dependsOn` empty → else 400.
- `GET /api/assignments/:id/edit` — resolve by ID.
- `PATCH /api/assignments/:id` — immutability: restore `id` and `project` from existing file via `setTopLevelField`.
- `PATCH /api/assignments/:id/plan` — update plan.
- `PATCH /api/assignments/:id/scratchpad` — update scratchpad.
- `POST /api/assignments/:id/handoff/entries` — append.
- `POST /api/assignments/:id/decision-record/entries` — append.
- `POST /api/assignments/:id/comments` — append comment (mirror of project-nested).
- `PATCH /api/assignments/:id/comments/:commentId/resolved` — toggle resolved on question.
- `POST /api/assignments/:id/transitions/:command` — call `executeTransitionByDir`.
- `POST /api/assignments/:id/status-override` — override.
- `PATCH /api/assignments/:id/acceptance-criteria/:index` — toggle criterion.

Duplicate-UUID prevention: every create path calls `resolveAssignmentById` first; on collision regenerate.

### 1.4 Backend — `src/dashboard/scanner.ts`, `watcher.ts`, `autodiscovery.ts`
- `loadWorkspaceRecords(projectsDir, assignmentsDir)` — scan both.
- `watcher.ts`: add `assignmentsDir?: string` to `WatcherOptions`; if set and exists, create additional watcher, broadcast `assignment-updated` with `projectSlug: null` on standalone changes.
- `autodiscovery.ts`: thread `assignmentsDir` through.

### 1.5 Backend — `src/dashboard/agent-sessions.ts`, `api-agent-sessions.ts`
- `reconcileActiveSessions` — drop the `project_slug IS NOT NULL` filter. When null, resolve path via `assignmentsDir/<assignment_slug>`.
- Add `listSessionsByAssignment(resolved: ResolvedAssignment)` with SQL branching on `standalone`.
- New route: `GET /api/assignments/:id/sessions`.

### 1.6 Frontend — `dashboard/src/App.tsx`
Add routes (and `/w/:workspace` siblings):
- `/assignments/:id`
- `/assignments/:id/edit`
- `/assignments/:id/plan/edit`
- `/assignments/:id/scratchpad/edit`
- `/assignments/:id/handoff/edit`
- `/assignments/:id/decision-record/edit`

### 1.7 Frontend — `dashboard/src/hooks/useProjects.ts`
- Add `useAssignmentById(id)`, `useAssignmentSessionsById(id)`.
- Add `runAssignmentTransitionById(id, action, reason?)`, `overrideAssignmentStatusById(id, status)` in `dashboard/src/lib/assignments.ts`.

### 1.8 Frontend — pages
- `AssignmentDetail.tsx`, `AssignmentsPage.tsx`, `EditAssignment.tsx`, `EditAssignmentPlan.tsx`, `EditAssignmentScratchpad.tsx`, `AppendAssignmentHandoff.tsx`, `AppendAssignmentDecisionRecord.tsx`, `AgentSessionsPage.tsx`, `ServersPage.tsx`: accept `{ slug?, aslug?, id? }` from `useParams`. When `id` present → standalone URLs + hooks; otherwise project-nested.

### 1.9 Types — `src/dashboard/types.ts` and `dashboard/src/types.ts`
